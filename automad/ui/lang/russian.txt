sys_cache_info: Кэширование необходимо для ускорения работы вашего сайта, и его обычно следует включать. Однако при некоторых обстоятельствах вы можете захотеть отключить или очистить кеш.

-

sys_cache_clear: Очистить кэш

-

sys_cache_clear_info: Очистка кеша сайта приводит к повторному сканированию обновленных страниц и файлов. Это действие не удалит файлы страниц или изображения из каталога кеша.

-

sys_cache_monitor: Интервал проверки составляет

-

sys_cache_monitor_info: Когда кэширование включено, Automad будет регулярно проверять наличие изменений в файловой системе. Здесь можно определить частоту этих проверок.

-

sys_cache_lifetime: Время жизни кеша

-

sys_cache_lifetime_info: Все кэшированные данные и страницы Automad имеют максимальный срок службы. По истечении срока жизни объекта его кэш будет перестроен по запросу.

-

sys_cache_purge: Очистить каталог кеша

-

sys_cache_purge_info: В отличие от этого, чтобы просто очистить кеш, продувка каталога кэша также переместит все временные файлы, удаленные страницы, обновленные резервные копии и кэшированные изображения во временный каталог системы. Это заставит полную перестройку изображений и кэш-памяти страницы, а также очистку всех удаленных страниц. В большинстве систем временный каталог будет очищен автоматически время от времени или на перезагрузке. Однако в некоторых системах временные каталоги должны быть очищены вручную.

sys_config: Конфигурационный файл

-

sys_config_info: Расширенные настройки могут управляться непосредственно редактированием файла конфигурации. Обратите внимание, что только опытные пользователи должны редактировать файл конфигурации, чтобы избежать повреждения вашей установки. Если что-то пойдет не так, пожалуйста, удалите файл конфигурации вручную с вашего сервера, чтобы восстановить настройки по умолчанию.

-

sys_config_edit: Редактировать файл конфигурации

-

sys_debug_info: Когда отладка включена, все процессы Automad будут регистрироваться в консоли вашего браузера. Отладка необходима только для разработки или устранения неполадок и должна быть отключена во всех остальных случаях.

-

sys_feed_info: Когда опция RSS Feed включена, Automad автоматически генерирует документ RSS-подачи и публикует его под URL-адресом, который отображается ниже, чтобы позволить пользователям следить за вашими новейшими сообщениями.

-

sys_feed_url: RSS-канал будет доступен по следующей ссылке. Вы можете добавить его в свою навигацию, добавив новую страницу и используя URL ниже значения для поля Redirect.

-

sys_feed_fields: Вы можете сортировать, добавлять или удалить поля, которые должны быть включены в RSS-канал, перетаскивая их на одну из двух областей ниже.

-

sys_feed_fields_info_used: Переместить элементы в эту область, чтобы добавить их в RSS-канал

-

sys_feed_fields_info_unused: Переместить элементы в эту область, чтобы удалить их из корма RSS

-

sys_headless_edit_info: Структура возвращаемых данных JSON контролируется шаблоном. Вы можете редактировать этот шаблон с помощью встроенного языка шаблонов Automad для настройки полей с помощью переменных, объектов и структур управления.

-

sys_headless_info: Активация безголового режима превращает Automad в систему управления контентом, полностью сосредоточенную на создании контента без уровня представления. Хотя панель мониторинга по-прежнему можно использовать для управления контентом, все данные и файлы доступны через API JSON только для чтения.

-

sys_language: Язык

-

sys_language_info: Для панели управления доступно несколько языков. Переводы создаются автоматически. Эта функция все еще находится на экспериментальной стадии, поэтому некоторые переводы могут быть неточными.

sys_status_feed_disabled: RSS Feed отключен

sys_update: Обновить

-

sys_update_available: Доступна новая версия Automad, готовая к установке. <br> Пожалуйста, сделайте резервную копию текущей установки перед обновлением!

-

sys_update_current_version: Ваша текущая установленная версия

-

sys_update_items: Обратите внимание, что обновление затронет следующие файлы и каталоги:

-

sys_update_not_required: Automad обновлен.

-

sys_update_progress: Подождите и не закрывайте окно во время обновления ...

-

sys_update_to: Обновить до версии

-

sys_update_windows_warning: Перед запуском обновления убедитесь, что все файлы вашей установки закрыты во внешних программах, чтобы избежать проблем с разрешениями!

sys_user: Пользователи

-

sys_user_info: Здесь вы можете найти всю информацию, связанную с вашей учетной записью и измените свое имя пользователя, пароль или адрес электронной почты. Чтобы иметь возможность сбросить пароль по электронной почте, убедитесь, что вы ввели Vaild и Active адрес электронной почты ниже.

-

sys_user_add: Добавить пользователя

-

sys_user_invite: Пригласите пользователя по электронной почте

-

sys_user_name: Имя пользователя

-

sys_user_password: Пароль

-

sys_user_repeat_password: Повторите пароль

-

sys_user_email: Эл. адрес

-

sys_user_alert_no_email: Вы еще не добавили допустимый адрес электронной почты на вашу учетную запись. Обратите внимание, что необходим действительный адрес электронной почты, чтобы иметь возможность сбросить пароль. Нажмите здесь, чтобы добавить электронное письмо сейчас.

-

sys_user_change_password: Изменить пароль

sys_user_current_password: текущий пароль

-

sys_user_new_password: новый пароль

sys_user_registered_info: Также возможно зарегистрировать других пользователей, чтобы позволить им редактировать контент.
